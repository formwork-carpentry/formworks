# Carpenter Docs

Project documentation is organized by purpose so the repository root stays focused on code and runtime configuration.

## Sections

- `docs/plans/` — long-range build plans and Scrum planning documents
- `docs/status/` — current milestone, release-gate, and remaining-work tracking
- `docs/security/` — security audit artifacts
- `docs/archive/` — superseded audit snapshots kept for historical reference
- `docs/EXAMPLES-SNIPPETS.md` — copy-paste snippets for common framework features
- `docs/JSDOC-STANDARDS.md` — package documentation and terminology conventions
