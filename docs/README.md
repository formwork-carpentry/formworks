# Carpenter Docs

Project documentation is organized by purpose so the repository root stays focused on code and runtime configuration.

## Overview

Carpenter is a modular TypeScript application framework inspired by Laravel, designed for rapid, scalable, and maintainable backend and fullstack development. This documentation covers architecture, usage, extension, and best practices for all core and optional packages.

## Getting Started

- See the main [README in the repo root](../../README.md) for quickstart, installation, and monorepo structure.
- Explore [examples/](../../examples/) for runnable apps and integration demos.
- Use [starters/](../../starters/) for ready-to-fork project templates.

## Documentation Structure

- **Plans:** Long-term vision, Scrum plans, and architectural blueprints.
- **Status:** Milestone tracking, release checklists, and remaining work.
- **Security:** Security audits, threat models, and compliance notes.
- **Archive:** Historical audits and superseded plans.
- **Examples & Snippets:** Copy-paste code for common patterns and features.
- **JSDoc Standards:** Unified documentation and terminology conventions.
